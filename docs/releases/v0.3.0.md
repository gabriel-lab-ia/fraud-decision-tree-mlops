# Release Notes v0.3.0

## Executive Summary

v0.3.0 hardens the fraud Decision Tree MLOps baseline for a more production-like
workflow. The release adds governed model artifacts, SHA-256 manifest validation,
dataset contracts, deterministic dataset fingerprints, train/validation/test
splitting, richer model metrics, validation gates, versioned API endpoints, batch
prediction, security posture checks, telemetry improvements, PSI drift helpers,
Docker hardening, CI quality gates, and public operational documentation.

This release is still a portfolio and engineering baseline. It uses synthetic data
by default and is not approved for autonomous financial decisions.

## Main Changes

- Added project version `0.3.0`.
- Added artifact schema `fraud-model-artifact-v1`.
- Added sidecar model manifest with SHA-256 integrity hash.
- Added artifact loading validation for schema and manifest hash.
- Added data schema `transaction-dataset-v1`.
- Added dataset contract checks for columns, duplicates, missing values, numeric
  validity, ranges, binary target, both classes, and minimum class count.
- Added deterministic dataset fingerprinting.
- Added stratified train, validation, and test split.
- Added sigmoid probability calibration.
- Added validation-based threshold selection with default `max_f1` strategy.
- Added richer evaluation metrics and baseline comparison.
- Added model validation gates for recall, F1, ROC-AUC, PR-AUC, false-negative rate,
  Brier score, and PR-AUC lift over dummy baseline.
- Added FastAPI `/v1` endpoints and deprecated compatibility aliases.
- Added batch prediction with configurable max batch size.
- Added liveness, readiness, version, metrics, telemetry, and monitoring endpoints.
- Added request IDs on API responses.
- Added telemetry hashing of transaction IDs in production.
- Added PSI drift helper.
- Added MkDocs navigation and operational documentation.
- Added Docker non-root runtime and `.dockerignore`.
- Added CI permissions, concurrency, docs build, typecheck, and security gates.

## Breaking Changes

No intentional breaking change is required for existing simple `/predict` clients:
legacy `/predict`, `/health`, `/ready`, and `/telemetry/recent` aliases remain
available and are marked deprecated.

Behavioral changes to note:

- New clients should use `/v1/predict` and `/v1/predict/batch`.
- Prediction responses now include `decision_threshold`, `request_id`, and artifact
  schema metadata.
- Production mode now refuses unsafe settings when API-key auth is disabled, API key
  is missing, or wildcard CORS is configured.
- Model readiness now depends on governed artifact loading and manifest validation
  when a manifest exists.
- Training outputs additional ignored reports and a model manifest.

## Security

- Production settings require `ENABLE_API_KEY_AUTH=true` and non-empty `API_KEY`.
- Production settings reject wildcard CORS origins.
- Prediction routes enforce `x-api-key` when API-key auth is enabled.
- Transaction IDs are hashed in production telemetry.
- API settings support redacted safe dumps.
- Docker runtime uses a non-root user.
- `.gitignore` and `.dockerignore` exclude generated datasets, model artifacts,
  local databases, reports, caches, and secrets.
- Bandit security scan is part of `make quality-gates`.

Known security limitation: sklearn/joblib artifacts rely on Python serialization.
Only artifacts created by trusted governed training should be loaded.

## Observability

- MLflow logs parameters, metrics, tags, model metadata, data-quality report,
  artifact manifest, and sklearn model.
- Telemetry remains fail-soft when MongoDB is unavailable.
- `/v1/telemetry/recent` returns recent prediction events or HTTP 503.
- `/v1/monitoring/summary` reports prediction count, fraud prediction count/rate,
  average risk score, and timestamp.
- `/metrics` exposes a minimal Prometheus-compatible API metadata gauge.
- `scripts/e2e_smoke_test.py` validates live readiness, single predictions, batch
  prediction, and metrics output.

## Data And Model Governance

- Dataset contract validation runs before training.
- Dataset fingerprint is recorded in data-quality reports, MLflow tags, and model
  artifact metadata.
- Training uses stratified train, validation, and test splits.
- Validation split is used for threshold selection.
- Test split is used for final metrics.
- Artifact metadata records model version, project version, feature order,
  calibration method, threshold, dataset fingerprint, schema versions, run ID, Git
  commit, Python version, scikit-learn version, and metrics summary.
- Validation gates are configured in `configs/train_config.yaml`.

## Upgrade Instructions

1. Update dependencies with `uv sync --frozen --extra dev`.
2. Review `.env.example` and configure production values if needed.
3. Retrain the model with `make train`.
4. Validate model gates with `make validate` or `make train-ci`.
5. Start the API with `make api` or Docker Compose.
6. Move new clients from deprecated `/predict` to `/v1/predict`.
7. For production-like deployments, set:

```bash
APP_ENV=production
ENABLE_API_KEY_AUTH=true
API_KEY=<strong-secret>
CORS_ALLOW_ORIGINS=https://your-app.example
```

## Known Limitations

- Synthetic data is not representative of real fraud behavior.
- No automated hyperparameter search is implemented.
- No scheduled drift job or alert routing is implemented.
- PSI drift helper exists, but production windowing and dashboard automation are not
  included.
- No full model registry promotion workflow is implemented.
- No rate limiting is implemented.
- No fairness analysis or real-data calibration review is included.
- Joblib artifact loading should be restricted to trusted artifacts.
